# Skills

覆盖完整开发闭环：

```text
想法
  ↓
切范围 - 01 feature-slice-skill
  ↓
定架构契约 - 02 agent-architecture-contract-skill
  ↓
生成实现计划 - 03 implementation-plan-skill
  ↓
AI 辅助编码
  ↓
测试验证 - 04 verification-skill
  ↓
代码审查 - 05 ai-code-review-skill
  ↓
面试沉淀 - 06 - interview-extraction-skill
```

1. Feature Slice Skill：
   目的：控制任务范围，确保 AI 生成的代码符合实际需求。
   实际应用：在开始开发时，先将每个任务拆分为小功能块，让 AI 专注于具体的代码生成任务。
2. Agent Architecture Contract Skill：
   目的：确保架构设计符合需求，特别是 LangGraph 工作流的设计和决策节点的实现。
   实际应用：在每个模块开发前，制定清晰的架构设计和契约，确保系统的可扩展性和模块化。
3. Implementation Plan Skill：
   目的：为每个开发任务生成清晰的实施计划，避免在开发过程中失控。
   实际应用：在 AI 编程前，先手动制定模块实现计划，确保开发过程有条不紊。
4. Verification Skill：
   目的：确保生成的代码符合预期，减少错误和问题。
   实际应用：为每个模块编写测试用例，确保生成的代码能够通过验证。
5. AI Code Review Skill：
   目的：确保代码质量，避免依赖 AI 完全生成代码。
   实际应用：定期对 AI 生成的代码进行审查，确保代码符合项目标准和质量要求。
6. Interview Extraction Skill：
   目的：帮助你总结开发过程中的关键技术点，准备面试时使用。
   实际应用：在每个阶段完成后，写下关键决策和技术细节，作为面试材料。
